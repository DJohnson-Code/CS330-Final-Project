# CS 330 Portfolio Reflection

For my CS 330 project I created a 3D desk setup scene with a desk, monitor, keyboard, mouse, plant, and wall background. When designing the scene I broke each object down into simpler shapes before coding it. For example, the monitor was made from boxes and a cylinder, and the plant used a cylinder and cone. This helped me think through the scene in smaller parts instead of trying to build everything at once.

My design process was mostly based on building the scene piece by piece. I started with the main object, then added textures, lighting, camera controls, and extra objects to make the scene feel more complete. That approach is something I can use in future work because it keeps larger projects more manageable.

For development, iteration was a big part of the process. I would add something, run the project, check how it looked, and then adjust the scale, position, or lighting. Over the milestones, I got better at organizing the scene and using helper functions instead of putting everything in one place.

Computer science helps me work toward my goal of becoming a software engineer because it teaches me how to solve problems in a structured way. Computational graphics is not my main career focus, but this course helped me understand transformations, textures, lighting, and how different parts of a program work together visually. Those skills can still apply to future school projects and professional work because they build stronger problem-solving and programming habits.
